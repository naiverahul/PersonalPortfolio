What is an MCP Server?

MCP stands for Modular Command Processor. It's a thin service layer that listens for tool requests from Claude or any LLM that supports tool use. It then executes external logic (like a web search) and returns the results back to the model.

How MCP Architecture Works

  1. The user sends a query to Claude (via CLI or Web UI).
  2. Claude evaluates the query and decides to use a tool (e.g., web search).
  3. Claude sends a tool-use request to the MCP server with parameters.
  4. MCP routes the request to the appropriate tool logic (like DuckDuckGo API).
  5. The response is returned and Claude continues the conversation with new context.

This architecture allows Claude to simulate reasoning and perform real-time information retrieval while remaining stateless on its own.
